**Block 1: Prod compose label patching (after Dockerfile ARG patches)**

```bash
# Patch Traefik prod labels for fpm-nginx / fpm-apache (FrankenPHP is shipped default)
if [[ "$SPIN_PHP_VARIATION" != "frankenphp" ]]; then
    line_in_file --action replace \
        --file "$project_dir/docker-compose.prod.yml" \
        'traefik.http.services.symfony.loadbalancer.server.port=' \
        '      - "traefik.http.services.symfony.loadbalancer.server.port=8080"'

    line_in_file --action replace \
        --file "$project_dir/docker-compose.prod.yml" \
        'traefik.http.services.symfony.loadbalancer.server.scheme=' \
        '      - "traefik.http.services.symfony.loadbalancer.server.scheme=http"'
fi
```

Important: The replacement strings have 6 spaces of indentation (`      - "..."`). This matches the YAML indentation under `deploy.labels:` in docker-compose.prod.yml. The `line_in_file --action replace` uses prefix matching on the search string.

**Block 2: APP_SECRET generation (unconditional)**

```bash
# Generate Symfony APP_SECRET (unconditional -- both new and init need it)
if [[ -f "$project_dir/.env" ]]; then
    APP_SECRET=$(openssl rand -hex 16)
    line_in_file --action replace \
        --file "$project_dir/.env" \
        'APP_SECRET=' \
        "APP_SECRET=${APP_SECRET}"
fi
```

CRITICAL: This block MUST be OUTSIDE the `SPIN_INSTALL_DEPENDENCIES` conditional. If placed inside, `spin init` users get an empty APP_SECRET and Symfony refuses to start. Guard with `[[ -f "$project_dir/.env" ]]` because `.env` may not exist for `init` on a fresh clone without Composer install.

Do NOT modify existing functionality. Keep the existing Dockerfile ARG patching, Composer install block, email patching, and git init sections unchanged.
