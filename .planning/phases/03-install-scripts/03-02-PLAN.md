**Section 4 — Main execution (in this exact order):**

**Step A: Clear screen:**
```bash
clear
```

**Step B: Patch Dockerfile ARG defaults using `line_in_file --action replace`:**
```bash
line_in_file --action replace \
    --file "$project_dir/$php_dockerfile" \
    'ARG PHP_VERSION=' \
    "ARG PHP_VERSION=\"${SPIN_PHP_VERSION}\""

line_in_file --action replace \
    --file "$project_dir/$php_dockerfile" \
    'ARG PHP_VARIATION=' \
    "ARG PHP_VARIATION=\"${SPIN_PHP_VARIATION}\""

line_in_file --action replace \
    --file "$project_dir/$php_dockerfile" \
    'ARG PHP_OS_SUFFIX=' \
    "ARG PHP_OS_SUFFIX=\"${PHP_OS_SUFFIX}\""
```

CRITICAL: Use `--action replace` (NOT `exact`). The `replace` action matches lines STARTING WITH the search term and replaces the ENTIRE line. This is correct for ARG lines. Using `exact` would do a substring match and could cause unintended replacements.

CRITICAL: The search terms are `'ARG PHP_VERSION='`, `'ARG PHP_VARIATION='`, `'ARG PHP_OS_SUFFIX='` — these are the LINE PREFIXES. The replacement includes the full line with the user-selected value in double quotes.

**Step C: Install Composer dependencies (guarded by SPIN_INSTALL_DEPENDENCIES):**
```bash
if [[ "$SPIN_INSTALL_DEPENDENCIES" == "true" ]]; then
    docker pull "$SPIN_PHP_DOCKER_INSTALLER_IMAGE"

    if [[ "$SPIN_ACTION" == "init" ]]; then
        echo "Re-installing composer dependencies..."
        docker compose run --rm --no-deps --build \
            -e COMPOSER_CACHE_DIR=/dev/null \
            -e "SHOW_WELCOME_MESSAGE=false" \
            php \
            composer install

        echo "Installing Spin..."
        docker compose run --rm --build --no-deps --remove-orphans \
            -e COMPOSER_CACHE_DIR=/dev/null \
            -e "SHOW_WELCOME_MESSAGE=false" \
            php \
            composer require serversideup/spin --dev
    else
        echo "Installing Spin..."
        docker run --rm \
            -v "$project_dir:/var/www/html" \
            --user "${SPIN_USER_ID}:${SPIN_GROUP_ID}" \
            -e COMPOSER_CACHE_DIR=/dev/null \
            -e "SHOW_WELCOME_MESSAGE=false" \
            "$SPIN_PHP_DOCKER_INSTALLER_IMAGE" \
            composer require serversideup/spin --dev
    fi
fi
```

Key differences from Laravel:
- `init` action: uses `docker compose run` (compose stack available for existing project)
- `new` action: uses `docker run` with `$project_dir` volume mount (compose stack not yet up)
- NO node dependency installation, NO database initialization, NO process_selections call

**Step D: Patch server contact email in Traefik prod config and .spin.yml:**
```bash
line_in_file --action exact --ignore-missing \
    --file "$project_dir/.infrastructure/conf/traefik/prod/traefik.yml" \

line_in_file --action exact --ignore-missing \
    --file "$project_dir/.spin.yml" \
    "changeme@example.com" \
    "$SERVER_CONTACT"
```

CRITICAL: Use `--ignore-missing` for BOTH files. The prod traefik.yml does not exist yet (Phase 4 deliverable). The `.spin.yml` is fetched by spin's action_init and may or may not contain the placeholder. `--ignore-missing` makes `exact` silently skip if the search text is not found.
