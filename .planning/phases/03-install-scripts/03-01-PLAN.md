Create `install.sh` at the repository root following the Laravel install.sh structure exactly, with these Symfony-specific changes:

**Shebang and setup:**
```bash
#!/bin/env bash
set -e
```

**Section 1 — Capture input arguments:**
```bash
symfony_framework_args=("$@")
```

**Section 2 — Default PHP Docker Image variables:**
- `SPIN_PHP_VERSION` defaults to `"8.3"` (not 8.5 like Laravel — Symfony template ships with 8.3 as the Dockerfile default)
- `SPIN_PHP_DOCKER_INSTALLER_IMAGE` defaults to `serversideup/php:${SPIN_PHP_VERSION}-cli`
- `SPIN_PHP_DOCKER_BASE_IMAGE` defaults to `serversideup/php:${SPIN_PHP_VERSION}-frankenphp` (NOT fpm-nginx-alpine like Laravel)
- No `SPIN_PHP_DOCKER_IMAGE_PREFIX` needed (Symfony template doesn't use it)

**Section 3 — spin_project_files array (Symfony-specific, no node/yarn):**
```bash
declare -a spin_project_files=(
    "vendor"
    "composer.lock"
    ".infrastructure"
    "docker-compose*"
    "Dockerfile*"
    "var"
)
```

**Section 4 — SERVER_CONTACT init and SPIN_PROJECT_DIRECTORY computation:**
- `SERVER_CONTACT=""` initialization
- Same logic as Laravel but using `symfony_framework_args` and default project name `"symfony"` (not `"laravel"`)
- MUST `export SPIN_PROJECT_DIRECTORY`

**Section 5 — Helper functions:**

`delete_matching_pattern()` — copy exactly from Laravel.

`display_destructive_action_warning()` — adapt from Laravel:
- Remove Laravel-specific bullet points about vite.config.js
- Change bullets to Symfony-appropriate: "We will be deleting and reinstalling dependencies based on your composer settings." and "We will attempt to automatically update your configuration files."
- No mention of node or database

`project_files_exist()` — copy exactly from Laravel.

`set_colors()` — copy exactly from Laravel (same RAINBOW, RED, GREEN, YELLOW, BLUE, DIM, BOLD, RESET).

`show_header()` — Use same RAINBOW ASCII art layout as Laravel's SPIN word art, but change the tagline from "Let's get Laravel launched!" to "Let's get Symfony launched!" per CONTEXT.md decision. Keep the `printf` pattern with `$RAINBOW` and `$RESET`.

`prompt_php_variation()` — follow Laravel structure but:
- Reorder variations array: `("frankenphp" "fpm-nginx" "fpm-apache")` (FrankenPHP FIRST, not second)
- Update descriptions:
  - frankenphp: `"FrankenPHP (Modern PHP app server, worker mode, HTTP/2 & HTTP/3)"`
  - fpm-nginx: `"PHP-FPM + NGINX (Traditional, widely adopted)"`
  - fpm-apache: `"PHP-FPM + Apache (.htaccess support)"`
- Default: `[[ -z "$SPIN_PHP_VARIATION" ]] && SPIN_PHP_VARIATION="frankenphp"` (NOT fpm-nginx)
- Remove any "(Latest)" label from variations — that only applies to PHP versions

`prompt_php_version()` — follow Laravel structure but:
- For `new` action, versions: `("8.5" "8.4" "8.3")` (no 8.2, 8.1, 8.0, 7.4)
- FrankenPHP filter already limits to 8.3+ (same as Laravel)
- Keep the FrankenPHP note: "Note: FrankenPHP requires PHP 8.3 or higher"

`prompt_php_os()` — follow Laravel structure exactly, but add the Alpine + FrankenPHP warning AFTER selection and BEFORE returning:
```bash
if [[ "$SPIN_PHP_VARIATION" == "frankenphp" && "$SPIN_PHP_OS" == "alpine" ]]; then
    echo ""
    echo "${BOLD}${YELLOW}Warning:${RESET}"
    echo "Alpine Linux uses musl libc which has a smaller default thread stack size."
    echo "This can cause crashes in FrankenPHP worker mode."
    echo "Consider using Debian for FrankenPHP in production."
    echo ""
    sleep 2
fi
```

`assemble_php_docker_image()` — Symfony version differs from Laravel:
- No `SPIN_PHP_DOCKER_IMAGE_PREFIX` in image tags
- MUST compute and `export PHP_OS_SUFFIX` (used by post-install.sh to patch Dockerfile ARG):
  - debian: `PHP_OS_SUFFIX=""`
  - alpine: `PHP_OS_SUFFIX="-alpine"`
- For debian: installer image is `serversideup/php:${SPIN_PHP_VERSION}-cli`, base image is `serversideup/php:${SPIN_PHP_VERSION}-${SPIN_PHP_VARIATION}`
- For alpine: installer image is `serversideup/php:${SPIN_PHP_VERSION}-cli-${SPIN_PHP_OS}`, base image is `serversideup/php:${SPIN_PHP_VERSION}-${SPIN_PHP_VARIATION}-${SPIN_PHP_OS}`
- Print assembled image: `echo "${BOLD}${BLUE}Docker Base Image:${RESET} $SPIN_PHP_DOCKER_BASE_IMAGE"`

**Section 6 — Main Spin Action Functions:**

`new()` — Symfony version:
```bash
new(){
  docker pull "$SPIN_PHP_DOCKER_INSTALLER_IMAGE"

  docker run --rm \
    -v "$(pwd):/var/www/html" \
    --user "${SPIN_USER_ID}:${SPIN_GROUP_ID}" \
    -e COMPOSER_CACHE_DIR=/dev/null \
    -e "SHOW_WELCOME_MESSAGE=false" \
    "$SPIN_PHP_DOCKER_INSTALLER_IMAGE" \
    composer --no-cache create-project \
        symfony/skeleton:"^7.4" \
        "${symfony_framework_args[@]}"

  init --force
}
```
CRITICAL: Mount `$(pwd)` not `$SPIN_PROJECT_DIRECTORY` — composer create-project creates the target directory. Mounting the project dir would make Docker create it as root.

`init()` — copy exactly from Laravel (same --force flag handling, same project_files_exist check, same display_destructive_action_warning call, same cleanup loop).

**Section 7 — Main execution (call order matters):**
```bash
set_colors
prompt_php_variation    # FIRST (per CONTEXT.md prompt order)
prompt_php_version      # SECOND (filtered by variation)
prompt_php_os           # THIRD
assemble_php_docker_image

SERVER_CONTACT=$(prompt_and_update_file \
    --title "Server Contact" \
    --details "Set an email contact who should be notified for Let's Encrypt SSL renewals and other system alerts." \
    --prompt "Please enter your email" \
    --output-only \
    --validate "email")

export SERVER_CONTACT

if type "$SPIN_ACTION" &>/dev/null; then
  $SPIN_ACTION
else
  echo "The function '$SPIN_ACTION' does not exist."
  exit 1
fi
```

**Key constraints:**
- Do NOT include any database selection, feature selection, JS package manager, PHP extensions, or GitHub Actions prompts — Symfony template has none of these
- Do NOT include `show_spin_pro_notice` — no pro/open-source distinction
- Do NOT include `SPIN_PHP_DOCKER_IMAGE_PREFIX` variable
- The `prompt_php_variation` call MUST come before `prompt_php_version` (variation → version → OS → email order)
