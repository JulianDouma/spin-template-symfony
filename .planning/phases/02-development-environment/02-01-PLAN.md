```bash
openssl req -x509 -nodes -newkey rsa:4096 -sha256 \
  -days 3650 \
  -keyout .infrastructure/conf/traefik/dev/certificates/local-dev-key.pem \
  -out .infrastructure/conf/traefik/dev/certificates/local-dev.pem \
  -subj "/C=US/ST=Local/L=Local/O=Symfony Dev/CN=localhost" \
  -addext "subjectAltName=DNS:localhost,IP:127.0.0.1"
```

The `-addext "subjectAltName=..."` flag is REQUIRED — modern browsers reject certs without SAN. Parameters: rsa:4096 key, 10-year expiry, no passphrase. These are committed to the repo (dev-only self-signed certs, not secrets).

2. Create `template/docker-compose.dev.yml` with this exact content:
```yaml
services:
  traefik:
    ports:
      - "80:80"
      - "443:443"
    networks:
      development:
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
      - ./.infrastructure/conf/traefik/dev/traefik.yml:/traefik.yml:ro
      - ./.infrastructure/conf/traefik/dev/traefik-certs.yml:/traefik-certs.yml
      - ./.infrastructure/conf/traefik/dev/certificates/:/certificates

  php:
    build:
      target: development
      args:
        USER_ID: ${SPIN_USER_ID}
        GROUP_ID: ${SPIN_GROUP_ID}
    environment:
      CADDY_SERVER_ROOT: /var/www/html/public
    volumes:
      - .:/var/www/html/
      - symfony_var:/var/www/html/var
    networks:
      - development
    depends_on:
      - traefik
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.symfony.rule=HostRegexp(`localhost`)"
      - "traefik.http.routers.symfony.entrypoints=web"
      - "traefik.http.services.symfony.loadbalancer.server.port=8080"
      - "traefik.http.services.symfony.loadbalancer.server.scheme=http"

networks:
  development:

volumes:
  symfony_var:
```

Key details:
- Docker socket mounted `:ro` (Traefik only reads container events)
- `CADDY_SERVER_ROOT: /var/www/html/public` explicitly sets document root for FrankenPHP (belt-and-suspenders — default is already this value per serversideup/php docs, but explicit is safer for a template)
- Bind mount `.:/var/www/html/` comes BEFORE named volume `symfony_var:/var/www/html/var` — order matters for overlay behavior. The named volume "wins" for the var/ subtree, giving native Docker storage performance for cache/log writes
- `vendor/` stays in the bind mount intentionally — IDE needs it for autocompletion/static analysis
- Traefik labels use `symfony` as router/service name (not `laravel`)
- `HostRegexp(\`localhost\`)` uses Traefik v3 syntax (NOT v2 `{catchall:.*}` style)
- `entrypoints=web` (HTTP) — Traefik handles HTTPS termination at its edge via the self-signed certs; the PHP container only speaks HTTP on port 8080
- All three PHP variations (frankenphp, fpm-nginx, fpm-apache) use identical labels in dev (port 8080, scheme http). The variation difference only matters for production (Phase 4)
- `USER_ID`/`GROUP_ID` use `${SPIN_USER_ID}` and `${SPIN_GROUP_ID}` env vars, matching Spin CLI conventions
- No Mailpit service — decided against in CONTEXT.md; will be documented as an optional add-on in README (Phase 4, DEV-05)
- No Node service — out of scope per CONTEXT.md
