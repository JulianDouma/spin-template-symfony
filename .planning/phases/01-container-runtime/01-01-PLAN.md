```sh
#!/bin/sh
# .infrastructure/entrypoint.d/10-cache-warmup.sh
#
# Warms the Symfony cache at container start. Running at startup (not build
# time) ensures all services (Redis, database, etc.) are reachable.
#
# IMPORTANT: Use /bin/sh, not /bin/bash — Alpine images do not include bash.

set -e

APP_BASE_DIR="${APP_BASE_DIR:-/var/www/html}"

if [ -f "$APP_BASE_DIR/bin/console" ]; then
    echo "Running Symfony cache:warmup..."
    php "$APP_BASE_DIR/bin/console" cache:warmup --env=prod
else
    echo "Symfony bin/console not found at $APP_BASE_DIR — skipping cache warmup"
fi
```

Critical details:
- Shebang MUST be `#!/bin/sh` (not `#!/bin/bash`) for Alpine compatibility
- MUST use `set -e` for fail-fast behavior
- MUST use `APP_BASE_DIR` variable with default `/var/www/html` (matches serversideup/php convention)
- MUST guard with `[ -f "$APP_BASE_DIR/bin/console" ]` check so it does not fail on development containers without Symfony installed yet
- File naming: `10-` prefix means it runs early in the entrypoint sequence (scripts execute in alphanumeric order)

After creating the file, verify the Dockerfile's COPY line matches the source path by confirming the Dockerfile contains `COPY --chmod=755 .infrastructure/entrypoint.d/ /etc/entrypoint.d/` which will copy this script into the image.
